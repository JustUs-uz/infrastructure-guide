Установка и настройка операционной системы

Роль: Администратор

Уровень: Начинающий

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Hyper-V по умолчанию недоступен на Windows 11 Home Edition, но его можно активировать с помощью следующих действий.

Предварительные требования

  • Включённая виртуализация на уровне BIOS (Intel VT-x или AMD-V).
  • Windows 11 Home Edition.

Действия:

  1. Перезагрузите компьютер и войдите в настройки BIOS/UEFI.
  2. Найдите настройку виртуализации:
    • Для Intel: VT-x, Intel Virtualization Technology.
    • Для AMD: AMD-V, SVM Mode.
  3. Включите виртуализацию и сохраните изменения.
  4. Перезагрузите компьютер.

Примечание: Доступ к BIOS/UEFI обычно осуществляется при загрузке компьютера с помощью нажатия клавиш F2, Del или других, в зависимости от модели вашей материнской платы.

Действия:

  1. Нажмите сочетание клавиш Win + X на клавиатуре.
  2. В открывшемся меню выберите Windows PowerShell (администратор) или Terminal (администратор).
  3. Подтвердите запрос контроля учётных записей, нажав Да.

Теперь PowerShell открыт с правами администратора.

Действия:

  1. В окне PowerShell выполните следующую команду:
    2. Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
  2. Дождитесь завершения процесса установки компонентов.
  3. Когда будет предложено перезагрузить компьютер, введите Y и нажмите Enter.

Возможные проблемы:

Ошибка: "Не удалось найти указанную функцию".

Решение: Убедитесь, что команда введена правильно и что виртуализация включена в BIOS. Также проверьте, что вы запускаете PowerShell от имени администратора.

Чек-лист: Активирование Hyper-V

  • Виртуализация включена в BIOS/UEFI.
  • Hyper-V активирован через PowerShell.
  • Компьютер перезагружен после активации Hyper-V.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

В этом шаге мы создадим виртуальную машину с FreeBSD на вашем компьютере с Windows 11 Home Edition с помощью Hyper-V.

Предварительные требования

  • Активированный Hyper-V.
  • Скачанный ISO-образ FreeBSD (например, FreeBSD-14.1-RELEASE-amd64-disc1.iso).

Действия:

  1. Перейдите на официальный сайт FreeBSD: https://www.freebsd.org/where/
  2. Скачайте последний стабильный релиз FreeBSD в формате ISO.

Действия:

  1. Откройте Диспетчер Hyper-V:
    • Нажмите Пуск и начните вводить Hyper-V, затем выберите Диспетчер Hyper-V из результатов поиска.
  2. В правой панели выберите Новая > Виртуальная машина.
  3. Следуйте мастеру создания виртуальной машины:
    • Имя: Укажите название виртуальной машины, например, FreeBSD.
    • Поколение: Выберите Поколение 1.
    • Память: Установите минимум 4096 МБ (рекомендуется 8192 МБ).
    • Сеть: Выберите виртуальный коммутатор с доступом к интернету.
    • Жёсткий диск: Создайте новый виртуальный жёсткий диск объёмом не менее 64 ГБ.
    • Установка ОС: Выберите вариант установки с загрузочного CD/DVD-ROM и укажите путь к скачанному ISO-образу.
  4. Завершите создание виртуальной машины и запустите её.

Примечание: FreeBSD лучше совместима с виртуальными машинами поколения 1 в Hyper-V. Поколение 2 использует UEFI и может вызвать проблемы совместимости.

Действия:

  1. Запустите виртуальную машину. Появится меню установщика FreeBSD.
  2. Следуйте инструкциям установщика:
    • Выберите язык и раскладку клавиатуры.
    • Выберите Install для начала установки.
    • Укажите имя хоста (например, freebsd-vm).
    • Выберите компоненты для установки (можно оставить по умолчанию).
    • Настройте разделы диска, выбрав Auto (UFS).
    • Установите пароль для пользователя root.
    • Настройте сеть (обычно автоматическое обнаружение через DHCP).
    • Выберите часовой пояс.
    • Настройте службы (можно оставить по умолчанию).
    • Создайте пользователя с правами администратора.

Добавление пользователя в группу wheel

При создании пользователя выполните следующие шаги:

  1. Введите имя пользователя (например, admin).
  2. Установите пароль пользователя.
  3. Когда будет предложено «Invite user into other groups?», введите wheel для добавления пользователя в административную группу.
  4. Остальные настройки можно оставить по умолчанию или настроить по своему усмотрению.

Примечание: Группа wheel позволяет пользователю выполнять команды с привилегиями суперпользователя с использованием su или sudo.

Что делать, если забыли добавить пользователя в группу wheel?

Решение: После установки войдите под пользователем root и выполните команду:

pw groupmod wheel -m yourusername

Замените yourusername на имя вашего пользователя.

После завершения установки выберите Exit и перезагрузите систему.

Чек-лист: Создание виртуальной машины и установка FreeBSD

  • Виртуальная машина создана и запущена в Hyper-V.
  • FreeBSD установлена и доступна для входа.
  • Создан пользователь с правами администратора.

Полезные ссылки

Описание

После установки FreeBSD необходимо выполнить начальную настройку системы и установить дополнительные пакеты.

Перед началом работы рекомендуется обновить системные пакеты.

Действия:

  1. Войдите в систему под пользователем root.
  2. Выполните команду обновления списка пакетов:
    3. pkg update
  3. Обновите установленные пакеты:
    4. pkg upgrade

Объяснение команд

  • pkg update: Обновляет список доступных пакетов.
  • pkg upgrade: Обновляет установленные пакеты до последних версий.

Установим текстовый редактор nano для удобного редактирования файлов и другие необходимые утилиты.

Действия:

  1. Установите редактор nano:
    2. pkg install nano
  2. Установите другие полезные утилиты (опционально):
    3. pkg install bash htop wget git

Объяснение команд

  • pkg install: Устанавливает указанные пакеты.
  • bash: Альтернативная оболочка командной строки.
  • htop: Утилита для мониторинга системы.
  • wget: Инструмент для скачивания файлов из интернета.
  • git: Система контроля версий.

По умолчанию команда sudo не установлена и не настроена в FreeBSD. Мы установим sudo и настроим пользователя для использования этой команды.

Действия:

  1. Установите пакет sudo:
    2. pkg install sudo
  2. Добавьте пользователя в группу wheel, если ещё не сделали этого:
    3. pw usermod admin -G wheel
  3. Настройте файл /usr/local/etc/sudoers для разрешения использования sudo пользователями группы wheel: 
    4. visudo
  4. В редакторе найдите строку: 
    # %wheel ALL=(ALL) ALL
    Удалите символ # в начале строки, чтобы раскомментировать её: 
    %wheel ALL=(ALL) ALL
  5. Сохраните изменения и выйдите из редактора.

Объяснение команд

  • visudo: Безопасный способ редактирования файла sudoers, проверяет синтаксис перед сохранением.
  • pw usermod: Модифицирует параметры пользователя.

Редактор vi установлен по умолчанию в FreeBSD. Хотя мы установили nano, иногда может потребоваться использовать vi, например, при работе с visudo.

Основные команды vi

  • Режимы: vi имеет два режима: командный и режим ввода текста.
  • Вход в режим ввода: Нажмите i для входа в режим ввода.
  • Выход из режима ввода: Нажмите Esc для возврата в командный режим.
  • Сохранение и выход: В командном режиме введите :wq и нажмите Enter.
  • Выход без сохранения: В командном режиме введите :q! и нажмите Enter.
  • Перемещение по тексту: Используйте клавиши стрелок или клавиши h (влево), j (вниз), k (вверх), l (вправо).

Совет: Если вам неудобно работать с vi, можно изменить редактор по умолчанию для команды visudo:

export EDITOR=nano
visudo

Теперь при запуске visudo будет использоваться редактор nano.

  • Пользователь может использовать команду sudo.
  • Вы знакомы с основными командами редактора vi.
