Мониторинг и обслуживание системы

Роль: Администратор

Уровень: Начинающий

Описание

В этом шаге мы рассмотрим, как отслеживать использование системных ресурсов, таких как процессор, память, дисковое пространство и сеть, используя встроенные и дополнительные инструменты в FreeBSD.

Команда top отображает информацию о процессах, загруженности системы и использовании ресурсов в реальном времени.

Действия:

  1. Откройте терминал на вашей виртуальной машине FreeBSD.
  2. Выполните команду:
    3. top
  3. Вы увидите обновляемую таблицу процессов и использование ресурсов.
  4. Чтобы выйти из режима top, нажмите клавишу Q.

Объяснение вывода

  • Load Average: Средняя нагрузка на систему за последние 1, 5 и 15 минут.
  • Processes: Общее количество запущенных процессов и их состояния.
  • CPU usage: Использование процессора системными и пользовательскими процессами.
  • Mem и Swap: Использование оперативной памяти и своп-раздела.
  • Ниже отображается список процессов с информацией о потреблении ресурсов.

Утилита htop предоставляет более удобный и наглядный интерфейс для мониторинга процессов и ресурсов.

Установка htop:

  1. Установите htop с помощью пакета:
    2. sudo pkg install htop

Действия:

  1. Запустите htop:
    2. htop
  2. Вы увидите цветную таблицу с использованием ресурсов и списком процессов.
  3. Для навигации используйте клавиши стрелок.
  4. Чтобы выйти из htop, нажмите клавишу F10 или Q.

Преимущества htop

  • Цветовое отображение информации для лучшего восприятия.
  • Возможность прокрутки списка процессов.
  • Удобные клавиши быстрого доступа для управления процессами.

Важно отслеживать использование дискового пространства, чтобы избежать переполнения диска.

Действия:

  1. Выполните команду для отображения использования файловой системы:
    2. df -h
  2. Вы увидите список файловых систем с информацией о размере, использовании и доступном пространстве.

Объяснение команды

  • df: Отображает информацию о файловых системах.
  • -h: Выводит информацию в удобном для чтения формате (человеко-читаемый вид).

Для проверки использования оперативной памяти можно использовать команду free.

Установка команды free:

  1. В FreeBSD команда free недоступна по умолчанию. Вместо неё используем команду vmstat или установим пакет procps:
    2. sudo pkg install procps

Действия:

  1. Выполните команду:
    2. free -h
  2. Вы увидите информацию об использовании памяти в удобном формате.

Объяснение команды

  • free: Показывает информацию об использовании оперативной памяти.
  • -h: Вывод в человеко-читаемом формате.

Для мониторинга сетевой активности можно использовать команду netstat и другие утилиты.

Действия:

  1. Просмотреть активные сетевые подключения:
    2. netstat -an
  2. Просмотреть статистику по сетевым интерфейсам:
    3. netstat -i
  3. Для более подробного мониторинга можно установить утилиту iftop:
    4. sudo pkg install iftop
  4. Запустить iftop для просмотра трафика в реальном времени:
    5. sudo iftop -i em0
    • -i em0: Указывает интерфейс для мониторинга (замените em0 на ваш интерфейс).

Объяснение команд

  • netstat: Показывает различные сетевые статистики.
  • iftop: Отображает сетевой трафик в реальном времени.

Чек-лист: Проверка мониторинга системных ресурсов

  • Вы умеете использовать команды top и htop для мониторинга процессов.
  • Вы можете проверять использование дискового пространства с помощью df -h.
  • Вы знаете, как проверить использование памяти.
  • Вы умеете мониторить сетевую активность с помощью netstat и iftop.

Системные журналы (логи) помогают отслеживать события в системе и выявлять проблемы. В FreeBSD для этой цели используется служба syslogd.

Системные журналы хранятся в каталоге /var/log/.

Действия:

  1. Перейдите в каталог с журналами:
    2. cd /var/log/
  2. Посмотрите список файлов журналов:
    3. ls -l
  3. Ознакомьтесь с основными файлами:
    • messages: Общие системные сообщения.
    • security: Сообщения, связанные с безопасностью.
    • maillog: Журналы почтовых сервисов.
  4. Просмотрите содержимое журнала messages:
    5. sudo tail -f /var/log/messages
  5. Чтобы остановить просмотр, нажмите Ctrl + C.

Объяснение команды

  • tail -f: Отображает последние строки файла и обновляет их в реальном времени.

Ротация журналов позволяет ограничить размер файлов журналов и сохранять только актуальные записи.

Действия:

  1. Файл конфигурации для ротации журналов находится по пути /etc/newsyslog.conf.
  2. Откройте файл для просмотра:
    3. sudo nano /etc/newsyslog.conf
  3. В файле указаны правила для каждого журнала, например: 
    /var/log/messages              644  7     100  *     J
    • 644: Права доступа к файлу.
    • 7: Максимальное количество ротаций.
    • 100: Максимальный размер файла в килобайтах.
    • *: Периодичность (звёздочка означает, что ротация происходит при достижении размера или по сигналу).
    • J: Сжимать ли архивы (J - сжимать с помощью bzip2).
  4. При необходимости вы можете изменить настройки под свои нужды.
  5. Сохраните изменения и закройте файл.
  6. Для принудительной ротации журналов выполните команду:
    7. sudo newsyslog

Объяснение команды

  • newsyslog: Утилита для управления ротацией журналов в FreeBSD.

  • Вы знаете, где находятся системные журналы.
  • Вы умеете просматривать и анализировать журналы.
  • Вы можете настроить ротацию журналов по своим потребностям.

Регулярное обновление системы и пакетов помогает поддерживать безопасность и стабильность системы.

Утилита pkg используется для управления пакетами в FreeBSD.

Действия:

  1. Обновите список доступных пакетов:
    2. sudo pkg update
  2. Обновите установленные пакеты до последних версий:
    3. sudo pkg upgrade
  3. Следуйте инструкциям на экране для завершения обновления.

Объяснение команд

  • pkg update: Обновляет локальный кэш пакетов.
  • pkg upgrade: Обновляет установленные пакеты.

Для обновления базовой системы FreeBSD используется утилита freebsd-update.

Действия:

  1. Проверьте доступные обновления системы:
    2. sudo freebsd-update fetch
  2. Если обновления доступны, установите их:
    3. sudo freebsd-update install
  3. При обновлении ядра или системных компонентов может потребоваться перезагрузка системы.

Объяснение команд

  • freebsd-update fetch: Проверяет наличие обновлений системы.
  • freebsd-update install: Устанавливает загруженные обновления.

Важно: Перед обновлением системы рекомендуется сделать резервную копию важных данных.

Чек-лист: Обновление системы

  • Вы умеете обновлять установленные пакеты с помощью pkg.
  • Вы знаете, как обновлять базовую систему с помощью freebsd-update.
  • Вы понимаете важность регулярного обновления системы для поддержания безопасности.

Для автоматизации задач в FreeBSD используется планировщик cron. Он позволяет запускать скрипты и команды по расписанию.

Действия:

  1. Откройте таблицу задач cron для редактирования:
    2. crontab -e
  2. В редакторе добавьте новую строку задания, например: 
    0 2 * * * /usr/local/bin/backup.sh
    • Это задание будет запускать скрипт backup.sh каждый день в 2:00 ночи.
  3. Сохраните изменения и выйдите из редактора.

Объяснение формата cron

  • Формат строки задания: минуты часы день_месяца месяц день_недели команда.
  • Значение * означает "в любое значение этого поля".

Пример:

  • */5 * * * * /path/to/script.sh: Запускать скрипт каждые 5 минут.
  • 0 0 * * 0 /path/to/weekly.sh: Запускать скрипт каждое воскресенье в полночь.

Действия:

  1. Просмотреть текущие задания cron:
    2. crontab -l
  2. Удалить все задания cron для текущего пользователя:
    3. crontab -r
  3. Управлять заданиями других пользователей (требуются права root):
    4. sudo crontab -u username -e
    • Замените username на имя пользователя.

Объяснение команд

  • crontab -e: Редактирует задания cron для текущего пользователя.
  • crontab -l: Отображает текущие задания.
  • crontab -r: Удаляет все задания cron для текущего пользователя.

Чек-лист: Планирование задач с помощью cron

  • Вы умеете создавать и редактировать задания cron.
  • Вы понимаете формат расписания cron.
  • Вы можете просматривать и управлять заданиями cron для пользователей.

Описание

Резервное копирование важно для предотвращения потери данных. В этом шаге мы рассмотрим базовые методы создания резервных копий в FreeBSD.

Действия:

  1. Создайте каталог для хранения резервных копий:
    2. sudo mkdir /backup
  2. Создайте скрипт backup.sh:
    3. sudo nano /usr/local/bin/backup.sh
  3. Добавьте следующий контент в скрипт: 
    #!/bin/sh
tar -czvf /backup/home_$(date +%F).tar.gz /home
    • Скрипт создаёт архив каталога /home с текущей датой в имени файла.
  4. Сохраните и закройте файл.
  5. Сделайте скрипт исполняемым:
    6. sudo chmod +x /usr/local/bin/backup.sh
  6. Запустите скрипт вручную для проверки:
    7. sudo /usr/local/bin/backup.sh
  7. Проверьте, что файл резервной копии создан в каталоге /backup:
    8. ls -l /backup

Объяснение скрипта

  • #!/bin/sh: Указывает интерпретатор для скрипта.
  • tar -czvf: Команда для создания сжатого архива.
  • $(date +%F): Вставляет текущую дату в формате ГГГГ-ММ-ДД.

Теперь добавим наше задание в cron для автоматического выполнения.

Действия:

  1. Откройте таблицу задач cron:
    2. sudo crontab -e
  2. Добавьте строку для ежедневного запуска скрипта в 3:00 утра: 
    0 3 * * * /usr/local/bin/backup.sh
  3. Сохраните изменения и выйдите из редактора.

Теперь скрипт будет автоматически выполняться каждую ночь в 3:00.

Чек-лист: Резервное копирование данных

  • Вы умеете создавать простые скрипты резервного копирования.
  • Вы можете автоматизировать задачи резервного копирования с помощью cron.
  • Вы понимаете важность регулярного резервного копирования данных.
