Подготовка среды разработки

Роль: Разработчик

Уровень: Начинающий

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

В этом шаге мы установим Visual Studio Code (VS Code) и необходимые плагины для разработки нашего проекта.

Предварительные требования

  • Доступ к интернету.
  • Операционная система Windows 11 Home Edition.

Действия:

  1. Перейдите на официальный сайт Visual Studio Code: https://code.visualstudio.com/
  2. Нажмите кнопку Download for Windows.
  3. Скачайте установочный файл для Windows (User Installer) 64-bit.
  4. После скачивания запустите установочный файл.
  5. Следуйте инструкциям установщика, отметив опцию «Добавить в PATH», если такая есть.
  6. По завершении установки запустите Visual Studio Code.

Действия:

Для удобной работы с проектом нам потребуются следующие плагины:

  • ESLint - проверка кода на ошибки и стиль.
  • Prettier - автоматическое форматирование кода.
  • Pug - поддержка синтаксиса Pug.
  • Sass - поддержка синтаксиса Sass/SCSS.
  • GitLens - интеграция с Git.
  • Live Server - запуск локального сервера для предпросмотра.

Установка плагинов:

  1. Откройте VS Code.
  2. Нажмите на иконку расширений на левой панели (иконка в виде квадрата).
  3. В поле поиска введите название плагина, например, ESLint.
  4. Найдите нужный плагин в результатах поиска и нажмите кнопку Install.
  5. Повторите шаги 3-4 для каждого плагина из списка.

Чек-лист: Проверка успешной установки VS Code и плагинов

  • Visual Studio Code установлен и запускается без ошибок.
  • Все необходимые плагины установлены и активированы.

Полезные ссылки

  • Marketplace расширений VS Code

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Node.js необходим для работы с JavaScript на стороне сервера и для использования различных инструментов разработки.

Предварительные требования

  • Установленный Visual Studio Code.

Действия:

  1. Перейдите на официальный сайт Node.js: https://nodejs.org/
  2. Скачайте рекомендованную версию LTS (Long Term Support) для Windows (64-bit).
  3. Запустите установочный файл и следуйте инструкциям установщика.
  4. В процессе установки отметьте опцию «Добавить в PATH», если она доступна.

Действия:

  1. Откройте командную строку Windows (нажмите Win+R, введите cmd, нажмите Enter).
  2. Введите команду для проверки версии Node.js:
    3. node -v
  3. Введите команду для проверки версии npm:
    4. npm -v
  4. Если версии отображаются без ошибок, установка прошла успешно.

Чек-лист: Проверка установки Node.js и npm

  • Команда node -v отображает версию Node.js.
  • Команда npm -v отображает версию npm.

Полезные ссылки

  • Документация Node.js
  • Документация npm

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Создадим структуру проекта, установим необходимые инструменты и загрузим проект на GitHub.

Предварительные требования

  • Установленные Node.js и npm.
  • Установленный Git.
  • Аккаунт на GitHub.

Действия:

  1. Перейдите на официальный сайт Git: https://git-scm.com/download/win
  2. Скачайте установщик для Windows и запустите его.
  3. Следуйте инструкциям установщика, оставив настройки по умолчанию.
  4. После установки перезагрузите командную строку, если она была открыта.

Действия:

  1. Откройте командную строку или PowerShell.
  2. Перейдите в директорию, где вы хотите создать проект.
  3. Создайте папку проекта и перейдите в неё:
    4. mkdir {{PROJECT_NAME}}
cd {{PROJECT_NAME}}
  4. Инициализируйте npm-проект:
    5. npm init -y
  5. Создайте файл .gitignore и добавьте туда следующие строки:
    6. # Node modules
node_modules/

# Compiled output
dist/
build/

# Logs
logs/
*.log
npm-debug.log*

# OS files
.DS_Store
Thumbs.db

# Environment variables
.env
.env.local
.env.development.local
.env.test.local
.env.production.local

# IDE files
.idea/
.vscode/

# Miscellaneous
*.sublime-project
*.sublime-workspace

# TypeScript cache
*.tsbuildinfo

# Coverage directory used by testing tools
coverage/

# Temporary files
tmp/
temp/

# Build directories
docker/
kubernetes/

    Объяснение: Этот файл .gitignore отражает структуру вашего проекта и исключает директории и файлы, которые не нужно добавлять в репозиторий, включая каталоги node_modules, скомпилированные файлы, файлы конфигурации сред, временные файлы и директории сборки.

  6. Инициализируйте Git-репозиторий:
    7. git init
  7. Настройте ваше имя пользователя и email для Git:
    8. git config --global user.name "Ваше Имя"
git config --global user.email "you@example.com"

    Примечание: Эти данные будут использоваться для записи информации о том, кто сделал коммит. Если вы хотите установить эти данные только для текущего репозитория, уберите опцию --global.

  8. Создайте файл README.md с описанием проекта.
    9. echo "# {{PROJECT_NAME}}" > README.md
  9. Создайте структуру папок проекта согласно схеме:
    10. mkdir docker
mkdir kubernetes
mkdir src
mkdir src/client
mkdir src/server
mkdir src/server/routes
mkdir src/server/models
mkdir src/server/controllers
mkdir public
mkdir public/css
mkdir public/js
mkdir public/images

    Структура проекта:

    {{PROJECT_NAME}}/
├── docker/
├── kubernetes/
├── src/
│   ├── client/
│   ├── server/
│       ├── routes/
│       ├── models/
│       └── controllers/
├── public/
│   ├── css/
│   ├── js/
│   └── images/
├── .gitignore
├── package.json
├── README.md
  10. Добавьте все файлы в Git и сделайте первый коммит:
    11. git add .
git commit -m "Initial commit"

    Примечание: Если при выполнении команды git add . вы получаете предупреждение:

    warning: in the working copy of 'package.json', LF will be replaced by CRLF the next time Git touches it

    Это предупреждение связано с различиями в форматах переноса строк между Unix (LF) и Windows (CRLF). Чтобы настроить Git для автоматической обработки переносов строк, выполните команду:

    git config --global core.autocrlf true

    После этого предупреждение должно исчезнуть.

Действия:

  1. Перейдите на GitHub и войдите в свой аккаунт.
  2. Нажмите на кнопку New для создания нового репозитория.
  3. Введите имя репозитория (например, {{PROJECT_NAME}}), добавьте описание и установите его как Private или Public.
  4. Не создавайте README.md, .gitignore или лицензию — они у нас уже есть.
  5. Нажмите на кнопку Create repository.
  6. В разделе «…or push an existing repository from the command line» скопируйте команды: 
    7. git remote add origin https://github.com/yourusername/{{PROJECT_NAME}}.git
git branch -M main
git push -u origin main

    Замените yourusername на ваше имя пользователя на GitHub.

  7. Вставьте эти команды в командную строку и выполните их.
  8. При запросе введите ваши учетные данные GitHub или настройте SSH-ключи для аутентификации.

Действия:

  1. Установите пакеты как зависимости разработчика:
    2. npm install --save-dev typescript gulp gulp-pug gulp-sass sass gulp-typescript
  2. Установите необходимые пакеты как зависимости:
    3. npm install express
  3. Обновите файл package.json с необходимыми скриптами:
    4. "scripts": {
    "build": "gulp",
    "start": "node dist/server/index.js"
},
  4. Инициализируйте TypeScript, создав файл tsconfig.json:
    5. npx tsc --init
  5. Создайте файл gulpfile.js с задачами для компиляции Pug, Sass и TypeScript.

Чек-лист: Проверка инициализации проекта

  • Структура проекта создана согласно схеме.
  • Файл .gitignore содержит корректные исключения, соответствующие структуре проекта.
  • Проект инициализирован как Git-репозиторий.
  • Проект загружен на GitHub.
  • Установлены все необходимые пакеты и инструменты.

Дополнительная информация
