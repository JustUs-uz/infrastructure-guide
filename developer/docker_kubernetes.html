Docker и Kubernetes

Роль: Разработчик

Уровень: Начинающий

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

В этом шаге мы рассмотрим, что такое Docker, и почему его используют в современном разработке и деплое приложений.

Что такое Docker?

  • Docker — это платформа для разработки, доставки и запуска приложений в изолированных средах, называемых контейнерами.
  • Контейнеры позволяют упаковать приложение со всеми его зависимостями и обеспечить его работу в любой среде.

Преимущества использования Docker

  • Портативность: Контейнеры могут быть запущены на любой системе с установленным Docker.
  • Изоляция: Обеспечивает изолированную среду выполнения для приложений.
  • Скалируемость: Упрощает развертывание и масштабирование приложений.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Установим Docker Desktop на вашу локальную машину для работы с контейнерами.

Предварительные требования

  • Windows 11 Home Edition с включенной WSL 2 (Windows Subsystem for Linux).

Docker Desktop на Windows 11 Home Edition требует WSL 2 для работы.

Действия:

  1. Откройте PowerShell от имени администратора.
  2. Выполните команду для включения компонента WSL:
    3.     dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
  3. Включите компонент виртуализации:
    4.     dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
  4. Перезагрузите компьютер.
  5. После перезагрузки установите WSL 2 по умолчанию:
    6.     wsl --set-default-version 2
  6. Установите дистрибутив Linux (например, Ubuntu) из Microsoft Store.

Полезные ссылки:

  • Установка WSL на Windows 10 и 11

Действия:

  1. Перейдите на сайт Docker Desktop.
  2. Скачайте установщик для Windows.
  3. Запустите установщик и следуйте инструкциям.
  4. В процессе установки убедитесь, что опция Use WSL 2 instead of Hyper-V выбрана.
  5. После установки запустите Docker Desktop.
  6. Проверьте установку, выполнив команду в PowerShell или командной строке:
    7.     docker --version

Объяснение: Docker Desktop позволяет управлять контейнерами и образами Docker прямо с вашей машины Windows.

Чек-лист: Проверка установки Docker

  • Docker Desktop установлен и запущен.
  • Команда docker --version выводит версию Docker.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Создадим Dockerfile для контейнеризации нашего Node.js приложения.

Предварительные требования

  • Docker установлен и работает.
  • Проект готов к контейнеризации.

Действия:

  1. В корневой директории проекта создайте файл Dockerfile:
    2.     touch Dockerfile
  2. Откройте файл Dockerfile в текстовом редакторе и добавьте следующий код: 
    3.     # Базовый образ
    FROM node:14

    # Установка рабочей директории
    WORKDIR /app

    # Копирование package.json и package-lock.json
    COPY package*.json ./

    # Установка зависимостей
    RUN npm install

    # Копирование исходного кода
    COPY . .

    # Сборка проекта
    RUN npm run build

    # Открытие порта
    EXPOSE 3000

    # Команда для запуска приложения
    CMD ["npm", "start"]

    Объяснение Dockerfile

    • FROM node:14: Используем официальный образ Node.js версии 14.
    • WORKDIR /app: Устанавливаем рабочую директорию внутри контейнера.
    • COPY package*.json ./: Копируем файлы package.json и package-lock.json.
    • RUN npm install: Устанавливаем зависимости.
    • COPY . .: Копируем все файлы проекта в контейнер.
    • RUN npm run build: Собираем проект с помощью Gulp.
    • EXPOSE 3000: Открываем порт 3000 для доступа к приложению.
    • CMD ["npm", "start"]: Команда для запуска приложения.

Файл .dockerignore указывает Docker, какие файлы и директории игнорировать при сборке образа.

Действия:

  1. Создайте файл .dockerignore в корневой директории проекта:
    2.     touch .dockerignore
  2. Добавьте следующие строки в файл .dockerignore:
    3.     node_modules
    npm-debug.log
    Dockerfile
    docker-compose.yml
    .git
    .gitignore
    .github
    README.md
    dist

    Объяснение .dockerignore

    • node_modules: Не копировать папку с модулями Node.js (они будут установлены внутри контейнера).
    • Другие файлы и папки, которые не нужны внутри контейнера.

Чек-лист: Проверка Dockerfile

  • Файл Dockerfile создан и содержит корректные инструкции.
  • Файл .dockerignore создан и содержит необходимые исключения.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Соберём Docker-образ нашего приложения и запустим его в контейнере.

Действия:

  1. Откройте терминал в корневой директории проекта.
  2. Выполните команду для сборки образа:
    3.     docker build -t yourusername/yourapp:latest .

    Замените yourusername/yourapp на желаемое имя образа.

    Объяснение команды

    • docker build: Команда для сборки Docker-образа.
    • -t yourusername/yourapp:latest: Тегируем образ с указанным именем и тегом latest.
    • .: Указываем текущую директорию как контекст сборки.

Действия:

  1. Выполните команду для запуска контейнера:
    2.     docker run -p 3000:3000 yourusername/yourapp:latest

    Объяснение команды

    • docker run: Команда для запуска контейнера из образа.
    • -p 3000:3000: Пробрасываем порт 3000 из контейнера на порт 3000 локальной машины.
    • yourusername/yourapp:latest: Указываем имя образа для запуска.
  2. Откройте браузер и перейдите по адресу http://localhost:3000.
  3. Убедитесь, что приложение работает внутри контейнера.

Чек-лист: Проверка работы контейнера

  • Docker-образ успешно собран без ошибок.
  • Контейнер запускается и приложение доступно по адресу http://localhost:3000.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Рассмотрим, что такое Kubernetes, и как он помогает в управлении контейнеризированными приложениями.

Что такое Kubernetes?

  • Kubernetes — это система оркестрации контейнеров с открытым исходным кодом, разработанная Google.
  • Она позволяет автоматизировать развертывание, масштабирование и управление контейнеризованными приложениями.

Преимущества использования Kubernetes

  • Автоматическое масштабирование приложений.
  • Балансировка нагрузки и управление трафиком.
  • Самовосстановление контейнеров при сбоях.
  • Управление конфигурацией и секретами.

Описание

Включим Kubernetes в Docker Desktop для работы с кластерами локально.

Действия:

  1. Откройте Docker Desktop.
  2. Перейдите в настройки (иконка шестерёнки).
  3. Выберите раздел Kubernetes.
  4. Поставьте галочку Enable Kubernetes.
  5. Нажмите Apply & Restart.
  6. Дождитесь, пока Kubernetes будет установлен и запущен (это может занять несколько минут).

Объяснение: Docker Desktop позволяет запускать однузвёздочный кластер Kubernetes для разработки и тестирования.

Утилита kubectl используется для управления Kubernetes-кластером.

Действия:

  1. Откройте PowerShell или командную строку.
  2. Проверьте версию kubectl:
    3.     kubectl version --client
  3. Если kubectl не установлен, его можно установить с помощью следующих команд:
    4.     choco install kubernetes-cli

    Примечание: Эта команда использует Chocolatey пакетный менеджер. Убедитесь, что он установлен.

Чек-лист: Проверка Kubernetes

  • Kubernetes включен в Docker Desktop.
  • Команда kubectl version --client выводит версию клиента kubectl.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Создадим файлы манифестов Kubernetes для развертывания нашего приложения в кластер.

Действия:

  1. Перейдите в директорию kubernetes вашего проекта.
  2. Создайте файл deployment.yaml.
  3. Добавьте следующий код:
    4.     apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: my-app-deployment
    spec:
      replicas: 2
      selector:
        matchLabels:
          app: my-app
      template:
        metadata:
          labels:
            app: my-app
        spec:
          containers:
          - name: my-app-container
            image: yourusername/yourapp:latest
            ports:
            - containerPort: 3000

    Объяснение deployment.yaml

    • apiVersion: apps/v1: Версия API Kubernetes.
    • kind: Deployment: Тип объекта Kubernetes.
    • metadata: Метаданные объекта.
    • spec: Спецификация Deployment.
    • replicas: 2: Количество подов (Pods) для запуска.
    • selector: Указывает, какие поды управляются этим Deployment.
    • template: Шаблон для создания подов.

Действия:

  1. Создайте файл service.yaml в той же директории.
  2. Добавьте следующий код:
    3.     apiVersion: v1
    kind: Service
    metadata:
      name: my-app-service
    spec:
      type: LoadBalancer
      selector:
        app: my-app
      ports:
      - protocol: TCP
        port: 80
        targetPort: 3000

    Объяснение service.yaml

    • kind: Service: Тип объекта Kubernetes.
    • type: LoadBalancer: Сервис будет доступен извне через балансировщик нагрузки.
    • selector: Связывает сервис с подами по метке app: my-app.
    • ports: Указывает, что порт 80 на сервисе направляется на порт 3000 внутри пода.

Чек-лист: Проверка манифестов

  • Файлы deployment.yaml и service.yaml созданы и содержат корректный код.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

Развернём наше приложение в локальном Kubernetes-кластере.

Действия:

  1. Откройте терминал и перейдите в директорию kubernetes.
  2. Примените манифесты командой:
    3.     kubectl apply -f deployment.yaml
    kubectl apply -f service.yaml

Действия:

  1. Проверьте статус подов:
    2.     kubectl get pods
  2. Проверьте статус сервисов:
    3.     kubectl get services
  3. Найдите EXTERNAL-IP вашего сервиса (может быть указано как <pending>).
  4. Поскольку мы используем локальный кластер, можно получить доступ к сервису по NodePort или использовать port-forward:
    5.     kubectl port-forward service/my-app-service 8080:80
  5. Откройте браузер и перейдите по адресу http://localhost:8080.

Чек-лист: Проверка развертывания

  • Поды запущены и имеют статус Running.
  • Сервис доступен и приложение работает в Kubernetes.

На какой машине: Локальная машина (Windows 11 Home Edition)

Описание

После завершения работы важно очистить ресурсы Kubernetes.

Действия:

  1. Удалите ресурсы командой:
    2.     kubectl delete -f deployment.yaml
    kubectl delete -f service.yaml
  2. Проверьте, что ресурсы удалены:
    3.     kubectl get pods
    kubectl get services

Чек-лист: Проверка очистки ресурсов

  • Все созданные поды и сервисы удалены.
  • Кластер Kubernetes не содержит ваших ресурсов.
